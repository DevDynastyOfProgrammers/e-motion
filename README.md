Так как проект использует `uv` и содержит файлы, исключенные из git (веса моделей), простого `git clone` недостаточно.

#### Шаг 1: Клонирование и установка зависимостей

1.  **Склонируй репозиторий:**
    ```bash
    git clone <твоя-ссылка-на-репозиторий>
    cd e-motion
    ```

2.  **Установи зависимости через `uv`:**
    ```bash
    uv sync
    ```

#### Шаг 2: Настройка окружения (.env)

Создай файл `.env` в корне проекта (скопируй содержимое из .env.example).

#### Шаг 3: Веса модели зрения (Vision Model)

Веса моделей в репозитории не хранятся, поэтому файл `emotion_model.pth` находится в `.gitignore`.
*   **Действие:** Скопируй [этот](https://drive.google.com/file/d/1buFuHepfhPaJNBAPw6NIKkpbPjzBaxr8/view?usp=sharing) файл вручную с флешки/облака/старого ноутбука в папку `assets/models/` (имя обязательно `emotion_model.pth`).

#### Шаг 4: Генерация данных для "Игрового Директора" (State Model)

Прототипы состояний (`.npy` файлы) тоже игнорируются git, но их не нужно копировать — их можно сгенерировать скриптами.

Выполни команды по очереди:

1.  **Генерация синтетического датасета:**
    ```bash
    uv run python -m ml.state.data_pipeline.generator
    ```
2.  **Обучение (создание прототипов):**
    ```bash
    uv run python -m ml.state.model.trainer
    ```

#### Шаг 5: Запуск

Теперь всё готово.

```bash
uv run main.py
```
